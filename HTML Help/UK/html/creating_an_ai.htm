Creating an AI

You can alter the behaviour of the game’s AI (artificial intelligence) computer-simulated players by tinkering with the settings in MONOPOLY CD-ROM’s AI Definition files.

During the installation of MONOPOLY CD-ROM, an AI directory is created on your hard disk, containing files for all the predefined AI computer-simulated players. One of these files, called standard.ai, contains all of the decision variables that you can change. To create your own, customised computer-simulated opponent, copy standard.ai, giving the new file a name of your choice, such as myai.ai. Call the file up using a text editor, such as DOSEDIT: you can now change any or all of the variables.

The first line of the file must always be left as “[general]” as shown in standard.ai. Other than that, you may set as many decision variables as you want. Any variables you do not set will automatically revert to the MONOPOLY CD-ROM game's defaults.

The usual syntax for setting a decision variable is: decision_variable = number.

A number of key decision variables are described in this section. But to find out about other, hidden decision variables, visit the Westwood Studios web site at http://www.westwood.com. From the Home Page, click on Index and select MONOPOLY CD-ROM game Frequently Asked Questions (FAQ).

You can influence other aspects of the game by tweaking the following variables:

OPTIMISM

The range of values is 0 - 5. This decision variable allows the AI to predict how successful it will be in acquiring future properties in a bid to achieve a monopoly.

MONOPOLY_OPTIMISM

The range of values is 0 - 2. This decision variable is similar to OPTIMISM, but it allows the AI to predict its chances of actually achieving a monopoly, not just of acquiring properties in a bid to achieve a monopoly.

MAX_REMAINING_CIRCUITS

The range of values is 40 - 100. This allows the AI to predict how many circuits it thinks the game will last. The greater the value, the higher the AI will value properties when buying or selling.

AVERAGE_POUNDS_PER_CIRCUIT

The range of values is 0 - 1000. This decision variable allows the AI to predict by how much, on average, its wealth will increase each time it completes a single circuit around the board.

PROPERTY_MULTIPLIER

The range of values is 0 - 10. This variable raises the AI’s perceived value of property versus cash. The higher the value, the more the AI will value property over money.

SINGLE_STN_MULTIPLIER

The range of values is 0 - 10. This variable increases the AI’s perceived value of train stations relative to other properties.

MINIMUM_DESIRED_CASH_IN_HAND

The range of values is 0 - 1000. This variable instructs the AI to always try to keep the amount of cash you specify in hand. The higher the figure, the less aggressive the AI will be in purchasing property and buildings.

SEND_PROPERTY_TO_AUCTION_FACTOR

The range of values is 0 - 4. This variable determines how cunning your AI will be. When it lands on a property, it will look at other players’ cash. If other players are short of money, the AI will send the property to auction, and attempt to pay less than its printed price. The lower the setting, the lower the amount of money in other players’ hands that will trigger the AI into trying this crafty move.

SEND_PROPERTY_TO_AUCTION_PERCENTAGE

The range of values is 0 - 100. Based on the SEND_PROPERTY_TO_AUCTION_FACTOR, this decision variable will determine the likelihood of the AI sending the property to auction.

MINIMUM_PROFIT

The range of values is 0 - 2. The higher this variable is above 1.0, the more advantage your AI will require before accepting a deal.

DESIRED_PROFIT

The range of values is 0 - 2. This decision variable tells the AI how much advantage it should seek to gain from any deal. The setting should be equal to or greater than MINIMUM_PROFIT.

VALUE_OF_GET_OUT_OF_JAIL_FREE_CARD

The range of values is 0 - 60. This number tells your AI how much a Get Out of Jail Free card is worth for deal-making purposes.

CASH_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable increases the AI’s perception of the value of money for deal-making purposes.

IMMUNITY_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable increases the AI’s perception of the value of immunity for deal-making purposes.

NON_MONOPOLY_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable affects the AI’s perception of the value of properties that are not part of a monopoly. The higher the value, the more reluctant the AI will be to mortgage or sell properties to pay off a debt.

MONOPOLY_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable affects the AI’s perception of the value of properties that are part of a monopoly. The higher the value, the more reluctant the AI will be to mortgage or sell monopoly properties to pay off a debt.

HOUSES_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable affects the AI’s perception of the value of houses and hotels. The higher the value, the more reluctant the AI will be to sell buildings to pay off a debt.

PERCENTAGE_OF_TIME_LOOKS_FOR_TRADE

The range of values is 0 - 100. This variable determines how often the AI will seek a good bargain opportunity in a given period of time. The higher the setting, the more frequently it seeks.

MAX_AI_DECISION_DELAY

The range of values is 0 - 30 seconds. This is the longest length of time during which your AI will think about a deal.

MIN_AI_DECISION_DELAY

The range of values is 0 - 30 seconds. This is the shortest length of time during which your AI will think about a deal.

PROBABILITY_OF_BIDDING_HIGH

The range of values is 0 - 100. This decision variable increases the chance of your AI bidding in high increments for a property at an auction.

PROBABILITY_OF_BIDDING_LOW

The range of values is 0 - 100. This decision variable increases the chance of your AI bidding in low increments for a property at an auction.

MAX_ROLL_DICE_DELAY

The range of values is 0 - 20 seconds. This is the length of time your AI will wait before throwing the dice to play its turn.

OFFERS_TRADES

The range of values is 0 - 6. This tells your AI the maximum number of people to include in a deal. A value of 0 or 1 is tantamount to instructing the AI never to propose a deal.

ALWAYS_BUYS_PROPERTY

The range of values is 0 or 1. If set to 1, the AI will do anything it can to buy a vacant property upon which it lands.

ALWAYS_REFUSES_TRADE

The range of values is 0 or 1. If set to 1, the AI will never accept a deal.

ALWAYS_GETS_OUT_OF_JAIL

The range of values is 0 or 1. If set to 1, the AI will always try to get out of jail as quickly as possible by paying.

ALWAYS_STAYS_IN_JAIL

The range of values is 0 or 1. If set to 1, the AI will always try to stay in jail as long as possible.

NEVER_BIDS_OVER_PRINTED_PRICE

The range of values is 0 or 1. If set to 1, the AI will never bid over the list price for property at an auction.

BUILD_ONLY_HOUSES

The range of values is 0 or 1. If set to 1, the AI will, in a bid to cause a housing shortage, never build hotels.

NEVER_ACCEPTS_IMMUNITY

The range of values is 0 or 1. If set to 1, the AI will never accept immunity as part of a deal.

ALWAYS_ACCEPTS_MONOPOLY_IN_TRADE

The range of values is 0 or 1. If set to 1, the AI will always accept a deal if this will lead to it getting a monopoly.

LOW_CASH_MULTIPLIER

The range of values is 0 - 10. This variable affects your computer-simulated opponent (AI)'s behaviour when faced with a shortage of money. The higher the value, the more reluctant the AI will be to spend its dwindling cash reserves to pay off a debt, and the more likely it will be to resort to alternative assets.
