Creating an AI

You can create your own artificial intelligence (AI), through the MONOPOLY CD-ROM game AI definition files. By adjusting different decision variables of MONOPOLY CD-ROM game AI, it will affect the way the computer player makes decisions.

As part of the MONOPOLY CD-ROM Game installation, an AI directory is created on your hard drive containing all of the predefined AI computer players. One of these files, called standard.ai contains all of the decision variables that you can change. To create your own AI, copy standard.ai over to your own file name, such as myai.ai. Using your favorite editor, such as DOSEDIT, you may change any or all of the decision values.

The first line of the file must always be: [general] as shown in standard.ai. Below that, you may set as many or as few decision variables as you want. Any variables not included in the file will be set to the MONOPOLY CD-ROM game's default.

The general format for a decision variable is: decision_variable = number.

Many of the decision variables are described in this section. But to find out about other secret decision variables, visit the Westwood Studios web site at http://www.westwood.com. From the Home Page, click on Index and select MONOPOLY CD-ROM game Frequently Asked Questions (FAQ).

Explanation of the Decision Variables

The first set of twenty-eight decision variables lets you tell your AI how much it likes a particular property. The more your AI likes a property, the more highly it will value that property for trading and acquisition purposes. The value must be from 0 - 200. An example is: NEW_YORK_AVENUE = 150.

Other values that you can set affect specific aspects of the game. These are:

OPTIMISM

The range of values is 0 - 5. This decision variable allows the AI to predict how likely it will be in successfully acquiring future properties toward a monopoly.

MONOPOLY_OPTIMISM

The range of values is 0 - 2. This decision variable is similar to OPTIMISM, except that it allows the AI to predict how likely it will be in successfully acquiring a complete monopoly, not just properties toward a monopoly.

MAX_REMAINING_REVOLUTIONS

The range of values is 40 - 100. This allows the AI to predict how many revolutions it thinks the game will last. The higher the value, the more highly the AI will value properties as part of a trade.

AVERAGE_DOLLARS_PER_REVOLUTION

The range of values is 0 - 1000. Please note that these values use the American Monopoly game. For example, Railroads are worth 200 in the American game. This decision variable allows the AI to predict how much, on average, the increase of its cash worth each time it completes a single revolution.

PROPERTY_MULTIPLIER

The range of values is 0 - 10. This variable increases the overall value of properties when comparing them to cash. The higher the value, the more highly the AI will value property over money.

SINGLE_RR_MULTIPLIER

The range of values is 0 - 10. This variable increases the overall value of a single railroad relative to other properties.

MINIMUM_DESIRED_CASH_ON_HAND

The range of values is 0 - 1000. Please note that these values use the American Monopoly game. For example, Railroads are worth 200 in the American game. This variable tells the AI to always try and keep this much cash on hand. The higher the number, the less aggressive the AI will be in purchasing property and buildings.

SEND_PROPERTY_TO_AUCTION_FACTOR

The range of values is 0 - 4. This variable allows your AI to be sneaky. When it lands on a property, it will look at other player's cash. If other players are low on money, it will send a property to auction, and attempt to pay less than its printed price. The lower this number, the lower the amount of money all other players need for the AI to try this sneaky move.

SEND_PROPERTY_TO_AUCTION_PERCENTAGE

The range of values is 0 - 100. After the SEND_PROPERTY_TO_AUCTION_FACTOR is looked at, this decision variable determines the liklihood of the AI sending the property to auction.

MINIMUM_PROFIT

The range of values is 0 - 2. The higher this variable is above 1.0, the more advantage your AI will require before accepting a trade.

DESIRED_PROFIT

The range of values is 0 - 2. This decision variable tells the AI how much advantage it would like to make in a trade. This number should be equal to or greater than MINIMUM_PROFIT.

VALUE_OF_GET_OUT_OF_JAIL_FREE_CARD

The range of values is 0 - 60. Please note that these values use the American Monopoly game. For example, Railroads are worth 200 in the American game. This number tells your AI how much a Get Out of Jail Free card is worth for trading purposes.

CASH_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable increases the overall value of cash for trading purposes.

IMMUNITY_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable increases the overall value of immunity for trading purposes.

NON_MONOPOLY_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable effects the value of properties that are not part of a monopoly. The higher the value, the more reluctant the AI will be to mortgage or sell properties to pay a debt.

MONOPOLY_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable effects the value of properties that are part of a monopoly. The higher the value, the more reluctant the AI will be to mortgage or sell monopoly properties to pay a debt.

HOUSES_VALUE_MULTIPLIER

The range of values is 0 - 10. This variable effects the value of houses and hotels. The higher the value, the more reluctant the AI will be to sell buildings to pay a debt.

PERCENTAGE_OF_TIME_LOOKS_FOR_TRADE

The range of values is 0 - 100. This variable determines how often the AI will look for a good trade. The higher the number, the more frequently it looks.

MAX_AI_DECISION_DELAY

The range of values is 0 - 30 seconds. This is the longest length of time that your AI will think about a trade.

MIN_AI_DECISION_DELAY

The range of values is 0 - 30 seconds. This is the minimum length of time that your AI will think about a trade.

PROBABILITY_OF_BIDDING_HIGH

The range of values is 0 - 100. This decision variable increases the chance of your AI bidding in high increments for a property at auction.

PROBABILITY_OF_BIDDING_MIDDLE

The range of values is 0 - 100. This decision variable increases the chance of your AI bidding in lower increments for a property at auction.

MAX_ROLL_DICE_DELAY

The range of values is 0 - 20 seconds. This is the delay your AI will wait before rolling dice on its turn.

OFFERS_TRADES

The range of values is 0 - 6. This tells your AI the maximum number of people to include in a trade. A value of 0 or 1 tells the AI to never offer a trade.

ALWAYS_BUYS_PROPERTY

The range of values is 0 or 1. When set to 1, this tells the AI to do whatever it takes to buy an unowned property upon which it lands.

ALWAYS_REFUSES_TRADE

The range of values is 0 or 1. When set to 1, this tells the AI to never accept a trade.

ALWAYS_GETS_OUT_OF_JAIL

The range of values is 0 or 1. When set to 1, this tells the AI to always try and get out of jail quickly by paying.

ALWAYS_STAYS_IN_JAIL

The range of values is 0 or 1. When set to 1, this tells the AI to always try and stay in jail as long as possible.

NEVER_BIDS_OVER_PRINTED_PRICE

The range of values is 0 or 1. When set to 1, this tells the AI to bid more than the printed price for a property at auction.

BUILD_ONLY_HOUSES

The range of values is 0 or 1. When set to 1, this tells the AI to never build hotels, in the hopes of causing a building shortage.

NEVER_ACCEPTS_IMMUNITY

The range of values is 0 or 1. When set to 1, this tells the AI to never accept immunity as part of a trade.

ALWAYS_ACCEPTS_MONOPOLY_IN_TRADE

The range of values is 0 or 1. When set to 1, this tells the AI to always accept a trade if it gets a monopoly.

LOW_CASH_MULTIPLIER

The range of values is 0 - 10. This variable effects how the AI treats a low cash situation. The higher the value, the more reluctant the AI will be to spend this cash to resolve a debt, relative to other revenue sources.
